# Установка и обновление

## Требования

- [Claude Code](https://claude.com/claude-code)
- `bash`, `tar`, `curl` — есть в macOS и Linux из коробки
- Опционально: Playwright для браузерного QA (`npx playwright install chromium`);
  `uvx` для засеянного MCP serena; `npx` для context7

Самому Keel **не нужен ни один сервис в рантайме**: ни своего MCP-сервера, ни
демона, ни индекса, который надо строить. Ядро — это shell и markdown.

## Быстрый старт

**1.** Скачайте `keel_1.5.0.tgz` и `keel_1.5.0.tgz.sha256` из
[Releases](https://github.com/bogdanov-igor/keel/releases/latest) в папку проекта.

**2.** Откройте проект в Claude Code и скажите:

> Установи keel из архива в этой папке: сверь sha256, распакуй, запусти
> `keel/install.sh` и расскажи, что появилось.

**3.** Если проект жил под SkillForge — или под любой системой до этой — скажите:

> Почисти остатки старой системы и предложи переаудит.

Это вся установка. Claude сам сверит контрольную сумму, распакует, поставит и
отчитается; шаг очистки уводит машинерию предшественника в карантин (ничего не
удаляя) и заводит переаудит в `BACKLOG.md`.

## Установка вручную

Те же два пути, набранные руками. Установщик запускается ровно тот же — это не
другая ветка кода, а те же команды, только без посредника.

### Из архива

У вас рядом лежат два файла: `keel_1.5.0.tgz` и его сайдкар `.sha256`.

```sh
cd /path/to/project                 # сюда скопированы оба файла
shasum -c keel_1.5.0.tgz.sha256     # сначала целостность: ждём "OK"
tar -xzf keel_1.5.0.tgz
bash keel/install.sh                # без аргумента = ставим в текущую директорию
```

Распакованную папку `keel/` можно оставить в проекте (тогда обновление — это
повторный запуск `install.sh`) либо удалить. Если оставляете — добавьте `keel/`
и `.tgz` в `.gitignore`.

### Из репозитория-исходника

```sh
git clone https://github.com/bogdanov-igor/keel.git
bash keel/install.sh /path/to/project
```

## Что делает установщик

| Действие | Подробности |
|---|---|
| Ставит ядро | Копирует `bundle/.claude` как настоящую директорию (никогда симлинком — симлинки ломают пути к памяти и разрешение хуков). Существующий `.claude` предварительно уезжает в `.claude.bak.<timestamp>`. |
| Сохраняет ваши скиллы | Любая директория скилла, которой нет в поставке ядра, переносится из прежнего `.claude` — включая скиллы, лежавшие в наследном `skills/_user/` от SkillForge. Вы никогда не теряете скилл, который написали сами. |
| Засевает состояние проекта | Создаёт `memory/`, `stages/`, `BACKLOG.md`, `PARKED.md`, `OPS.md`, `keel.json`, `.mcp.json` — **только там, где их нет**. Существующее состояние проекта никогда не перезаписывается. |
| Защищает секреты | Добавляет `.secrets.env` и `.qa/` в `.gitignore`. |
| Проставляет версию | Пишет `.claude/VERSION` — его читает проверка обновлений. |
| Находит остатки | Сообщает о хвостах SkillForge и показывает на скилл `migrate`. **Ничего не двигает** — установщик не трогает ваши файлы. |

## Обновление

Проще всего — скачать новый архив в папку проекта и сказать Claude:

> Обнови keel из архива в этой папке.

Вручную — та же команда, что и при установке. Берёте свежую папку keel
(скачиваете релиз или делаете `git pull`) и запускаете:

```sh
cd /path/to/project
bash keel/install.sh
```

Файлы ядра заменяются. Состояние проекта — память, стейджи, бэклог, парковка,
ваши собственные скиллы — не трогается. Старые бэкапы `.claude.bak.*` можно
спокойно подчищать.

### Как вы узнаёте, что вышло обновление

Хук `SessionStart` сверяет `.claude/VERSION` со свежим релизом наверху и
печатает **одну строку**, если версия новее, — а потом молчит сутки (кэш в
`~/.cache/keel/`). Если у вас актуальная версия, он не печатает вообще ничего,
так что обычный случай стоит ноль токенов.

Сессию он не блокирует никогда: у сетевого вызова потолок 3 секунды, и любой
сбойный путь — оффлайн, рейт-лимит, неразбираемый ответ — завершается молча.

Выключается в `keel.json`:

```json
{
  "update_check": {
    "enabled": false,
    "repo": "bogdanov-igor/keel",
    "interval_hours": 24
  }
}
```

## Сборка архива (для мейнтейнеров)

```sh
bash build-archive.sh    # → dist/keel_<version>.tgz + .sha256
```

`build-archive.sh` внутрь архива не кладётся. Прежде чем отрапортовать об
успехе, он проверяет сам себя: распаковывает tgz во временную директорию,
проводит настоящую установку и проверяет контракт, засеянные файлы, права на
хуки, отметку версии, документацию, лицензию и то, что проверка обновлений
молчит на собственной версии. При провале рабочая область остаётся на месте —
для разбора.
